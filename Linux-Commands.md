## 🛠️ Работа с дисками и разделами (низкоуровнево)

| Команда | Что делает | Важно! |
|---------|------------|--------|
| `sudo fdisk /dev/sdX` | Редактор разделов (MBR). Внутри: `p` (показать), `d` (удалить), `n` (новый), `w` (записать) | Не забудь `w`, иначе изменения не сохранятся. |
| `sudo gdisk /dev/sdX` | То же для GPT (UEFI) | |
| `sudo mkfs.ext4 /dev/sdX1` | Форматирует раздел в ext4 | Все данные будут уничтожены. |
| `sudo mkswap /dev/sdX2` | Делает раздел swap-областью | После создания включи: `swapon`. |
| `sudo swapon /dev/sdX2` | Включает swap-раздел | Проверить: `swapon --show`. |
| `sudo wipefs -a /dev/sdX` | Стирает ВСЕ файловые системы и таблицу разделов с диска | Полная зачистка. Данные не восстановить. |
| `sudo dd if=/dev/zero of=/dev/sdX bs=512 count=1` | Затирает первый сектор (убивает таблицу разделов) | Мягче, чем `wipefs`. Тоже необратимо. |
# Работа с NVMe диском: отключение и монтирование

# Простое отключение NVMe-дисков в Linux
Самый простой и безопасный способ временно отключить NVMe-диск в Linux — воспользоваться командой управления устройством через sysfs.

## Как правильно отключить NVMe-диск?

#Подготовка.
1. Проверьте состояние диска, убедитесь, что он не примонтирован:
2. lsblk | grep nvme

Если указаны примонтированные разделы типа `/dev/nvme0n1p*`, снимите их:
sudo umount /dev/nvme0n1p*

## Способ №1: временное отключение через sysfs

Выполните команду для временного удаления устройства (до следующей перезагрузки):

echo 1 | sudo tee /sys/class/nvme/nvme0/device/remove

Эта команда даёт ядру сигнал забыть о данном устройстве, оно больше не будет доступно системе.

## Альтернатива: выгрузка модуля драйвера NVMe
Можно также попробовать убрать модуль драйвера NVMe целиком, отключив таким образом все диски сразу:
bsudo rmmod nvme

Однако учтите, что эта команда отключит *все* подключённые NVMe-диски.

## Что именно делают эти команды?
- `echo 1 > /sys/class/nvme/nvme0/device/remove`: приказывает ядру удалить определённый NVMe-диск.
- `rmmod nvme`: выгружает модуль драйвера NVMe, отчего перестанут видеться все диски.
## Важные моменты:
- После перезагрузки диск автоматически появится снова.
- Для отключения конкретного диска (если у вас несколько NVMe) пользуйтесь первым методом.
- Вторая команда отключает все NVMe одновременно.
## Постоянное отключение при загрузке
Если хотите совсем исключить определение NVMe-диска при загрузке (например, при переустановке ОС), добавьте параметры ядра:
Добавляем временно при загрузке (в меню GRUB нажмите 'e' и допишите):
nvme_core.default_ps_max_latency_us=0 pcie_aspm=off

Или добавляем навсегда (редактировать /etc/default/grub):
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash nvme_core.default_ps_max_latency_us=0"sudo update-grub

## Проверка результата

После отключения проверьте статус:
lsblk | grep nvme

Если всё прошло успешно, никаких результатов выдано не будет.

## Установка скачанного пакета
sudo apt install ./имя.deb

### Ошибка прав при установке `.deb`
Если возникает ошибка вида «файл недоступен для пользователя `_apt`, переместите файл в `/tmp`:**
sudo mv /home/user/пакет.deb /tmp/sudo apt install /tmp/пакет.deb



---

## Проблемы блокировки `APT` (`Could not get lock`)
При возникновении ошибки блокировки:
sudo kill <PID процесса-блокировщика>


Или проще всего:
sudo systemctl restart apt-daily.timer && sudo systemctl stop apt-daily.service

**Альтернативное простое решение:** Перезагрузитесь!

## 🔧 Восстановление загрузчика **GRUB** (после установки Linux Mint)
#### Шаги для восстановления:
1. Загрузитесь с Live-флешки Mint.
2. Определите нужные разделы диска:
   ```
   lsblk
   ```
3. Примонтируйте корневой раздел системы (например, `/dev/sda2`):
   ```
   sudo mount /dev/sda2 /mnt
   ```
4. Если используется UEFI, примонтируйте ESP (обычно `/dev/sda1`):
   ```
   sudo mkdir -p /mnt/boot/efi
   sudo mount /dev/sda1 /mnt/boot/efi
   ```
5. Примонтируйте необходимые системные директории:
   ```
   sudo mount --bind /dev /mnt/dev
   sudo mount --bind /proc /mnt/proc
   sudo mount --bind /sys /mnt/sys
   ```
6. Перейдите в среду `chroot`:
   ```
   sudo chroot /mnt
   ```
7. Установите заново GRUB:
   Для UEFI:
   ```
   grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=ubuntu --recheck
   ```
   Для BIOS (Legacy):
   ```
   grub-install --target=i386-pc /dev/sda
   ```
8. Обновите конфигурацию GRUB:
   ```
   update-grub
   ```
9. Выход и перезагрузка:
   ```
   exit
   sudo umount -R /mnt
   sudo reboot
   ```

## Дополнительные советы при ошибке «*No working init found*»
Проверьте правильность UUID корневого раздела в конфигурации GRUB. Вы можете исправить значение при загрузке ядра (нажав клавишу `e` в меню GRUB и добавив правильный UUID вручную).

## 🧠 Основные понятия Linux
| Понятие     | Значение                                |
|-------------|----------------------------------------|
| UEFI vs Legacy | Современный режим загрузки (ESP/FAT32) vs устаревший (MBR)       |
| GPT vs MBR  | Новый стандарт для больших дисков (до 128 разделов) vs традиционный (только 4 основных раздела) |
| GRUB        | Загрузчик Linux, конфигурируется скриптом `update-grub`.                  |
| chroot      | Изменение корня среды исполнения для исправления неисправной системы.          |
| ESP         | Специальный раздел для хранения загрузчиков (используется в UEFI).            |

## 🚫 Распространённые ошибки и их устранение
| Ошибка                        | Причина                     | Решение                                    |
|-------------------------------|----------------------------|--------------------------------------------|
| Device or resource busy       | Раздел занят процессом      | Отмонтируйте раздел принудительно: `umount -l /dev/sdX*` |
| Cannot find EFI directory     | Не примонтирована ESP-раздел | Примонтируйте ESP: `mount /dev/sdX1 /mnt/boot/efi` |
| E: Could not get lock         | Процесс APT заблокировал установку | Используйте `kill` или перезагрузите систему |
| Permission denied             | Нет прав на чтение файла    | Переместите файл в `/tmp` перед установкой |
| No working init found         | Неправильный UUID корневого раздела | Исправьте UUID в настройках GRUB |
| Отсутствие прав на запуск скрипта | Файл не является исполняемым | Сделайте файл исполняемым: `chmod +x file.sh` |
