# Шпаргалка по Git (боевая, проверенная)

## 📁 Основные понятия

| Понятие | Что это |
|---------|---------|
| Репозиторий (repo) | Папка, в которой Git отслеживает изменения. Может быть локальным (на твоём компьютере) или удалённым (на GitHub). |
| Коммит (commit) | Снимок состояния файлов на момент времени. Имеет уникальный хеш (например, `a0583a3`). |
| Ветка (branch) | Независимая линия разработки. `main` (или `master`) — основная ветка. |
| HEAD | Указатель на текущий коммит (на каком месте мы сейчас находимся). |
| Удалённый репозиторий (remote) | Ссылка на репозиторий, расположенный на сервере (GitHub, GitLab и т.д.). Обычно называется `origin`. |

## 🔧 Настройка (первый раз)

| Команда | Что делает | Пример |
|---------|------------|--------|
| `git config --global user.name "Имя"` | Установить имя для коммитов | `git config --global user.name "AnderSunI"` |
| `git config --global user.email "email@example.com"` | Установить email | `git config --global user.email "anderson@example.com"` |
| `git config --global core.editor "nano"` | Установить редактор по умолчанию | `git config --global core.editor "nano"` |
| `git config --list` | Показать все настройки | |

## 🌱 Инициализация и клонирование

| Команда | Что делает |
|---------|------------|
| `git init` | Создать новый локальный репозиторий в текущей папке |
| `git clone <URL>` | Скопировать удалённый репозиторий к себе | `git clone https://github.com/AnderSunI/dnd-trader.git` |
| `git remote -v` | Показать, какие удалённые репозитории настроены |

## 📝 Работа с изменениями (локально)

| Команда | Что делает | Частые аргументы/Пример |
|---------|------------|-------------------------|
| `git status` | Показать, какие файлы изменены / добавлены / готовы к коммиту | |
| `git add <файл>` | Добавить файл в индекс (staging area) — подготовить к коммиту | `git add index.html` |
| `git add .` | Добавить **все** изменённые файлы в индексе (кроме тех, что в `.gitignore`) | |
| `git add -A` | Добавить всё, включая удалённые файлы | |
| `git commit -m "Сообщение"` | Сделать коммит (сохранить изменения) с коротким сообщением | |
| `git commit` | Откроет редактор для подробного сообщения | |
| `git commit -am "Сообщение"` | Добавить все отслеживаемые файлы и сразу закоммитить (не для новых файлов) | |
| `git diff` | Показать изменения, которые ещё не добавлены в индекс | |
| `git diff --staged` | Показать изменения, которые уже добавлены в индекс, но ещё не закоммичены | |
| `git reset <файл>` | Убрать файл из индекса (оставив изменения в рабочей папке) | |
| `git reset --hard HEAD` | Отменить все незакоммиченные изменения (ОСТОРОЖНО — потеряете правки) | |
| `git restore <файл>` | Откатить файл к последнему коммиту (похоже на `reset --hard` для одного файла) | |

## 🌿 Ветки

| Команда | Что делает | Пример |
|---------|------------|--------|
| `git branch` | Показать список веток (текущая подсвечена) | |
| `git branch <имя>` | Создать новую ветку | `git branch feature/filters` |
| `git checkout <ветка>` | Переключиться на другую ветку | `git checkout main` |
| `git switch <ветка>` | То же, что checkout (более новый способ) | |
| `git checkout -b <ветка>` | Создать ветку и сразу переключиться на неё | `git checkout -b new-branch` |
| `git merge <ветка>` | Влить изменения из указанной ветки в текущую | `git merge feature/filters` |
| `git branch -d <ветка>` | Удалить ветку (локально) | `git branch -d feature/filters` |
| `git push origin --delete <ветка>` | Удалить ветку на удалённом репозитории | |

## ☁️ Работа с удалённым репозиторием (GitHub)

| Команда | Что делает |
|---------|------------|
| `git push` | Отправить локальные коммиты в удалённый репозиторий (в текущую ветку) |
| `git push -u origin main` | Отправить ветку `main` и установить её как upstream (чтобы потом писать просто `git push`) |
| `git pull` | Забрать изменения из удалённого репозитория и объединить (fetch + merge) |
| `git fetch` | Забрать изменения, но не объединять (посмотреть, что изменилось) |
| `git clone <URL>` | Клонировать репозиторий (уже было) |

## 🔁 Просмотр истории

| Команда | Что делает |
|---------|------------|
| `git log` | Показать историю коммитов (выход — `q`) |
| `git log --oneline` | Краткий вид: хеш и сообщение |
| `git log --graph` | Показать ветвление в виде ASCII-графа |
| `git log --oneline --graph --all` | Кратко и с ветками |
| `git show <хеш>` | Показать изменения в конкретном коммите |

## ⚠️ Типичные проблемы и их решение

| Проблема | Причина | Решение |
|----------|---------|--------|
| `error: failed to push some refs` | В удалённом репозитории есть изменения, которых нет у тебя локально | Сначала `git pull`, потом `git push` |
| `Permission denied (publickey)` | Не настроен SSH-ключ или используется HTTPS без токена | Либо настроить SSH, либо использовать токен. В HTTPS: при запросе пароля вводить токен (не пароль от GitHub) |
| `Merge conflict` | Один и тот же файл изменён в двух ветках по-разному | Нужно вручную разрешить конфликт в файле, затем `git add` и `git commit` |
| `Your branch is ahead of 'origin/main' by X commits` | Есть локальные коммиты, которые не отправлены | `git push` |
| `You have unmerged paths` | Конфликт не разрешён до конца | Решить конфликт, добавить файлы и закоммитить |
| `Could not read from remote repository` | Проблема с сетью или аутентификацией | Проверить URL: `git remote -v`. Проверить, что есть доступ к интернету и правильные credentials |
| `fatal: refusing to merge unrelated histories` | Попытка слить ветки, у которых нет общего предка | Добавить флаг `--allow-unrelated-histories` (например, при `git pull`) |

## 🧹 Отмена и исправление

| Команда | Что делает |
|---------|------------|
| `git reset --soft HEAD~1` | Отменить последний коммит, но оставить изменения в индексе (можно поправить и снова закоммитить) |
| `git reset --mixed HEAD~1` | Отменить коммит, изменения останутся в рабочей папке, но убрать из индекса |
| `git reset --hard HEAD~1` | Отменить коммит **и уничтожить изменения** (осторожно!) |
| `git commit --amend` | Добавить изменения в последний коммит (исправить сообщение или добавить забытые файлы) |
| `git revert <хеш>` | Создать новый коммит, который отменяет изменения указанного коммита (безопаснее, чем reset) |

## 📦 Работа с токенами и SSH

### HTTPS с токеном
1. Создай токен на GitHub: Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token.
2. Скопируй токен (например, `ghp_XXXX`).
3. При `git push` введи имя пользователя `AnderSunI` и пароль — токен.
4. Чтобы не вводить каждый раз, можно сохранить токен в URL:
   ```bash
